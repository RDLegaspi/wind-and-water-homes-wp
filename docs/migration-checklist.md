# Migration Checklist

## Discovery

- Inventory all public URLs from the current site.
- Capture page titles, meta descriptions, headings, and canonical URLs.
- List forms, form recipients, automations, and thank-you pages.
- List custom content types, repeated sections, galleries, downloads, and embeds.
- Export or archive current media assets.
- Capture analytics, pixels, tag manager, and search console requirements.

## Build

- Confirm theme type: classic, block, or child theme.
- Create templates for home, standard pages, blog/archive, single posts, and 404.
- Define reusable fields or blocks for repeated content.
- Configure navigation menus and footer content.
- Add accessibility basics: semantic headings, alt text, focus states, and contrast.

## Content

- Import or recreate pages and posts.
- Optimize images before uploading.
- Recreate forms and test notifications.
- Add SEO metadata and social sharing images.
- Confirm internal links use final URLs.

## Launch

- Create a redirect map from old URLs to new URLs.
- Confirm SSL, permalink structure, sitemap, and robots settings.
- Verify analytics and search console.
- Test forms on staging and production.
- Run performance checks on key templates.
- Confirm backup and rollback plan.

## Post-Launch

- Crawl the live site for 404s and redirect errors.
- Submit sitemap.
- Monitor form submissions, analytics, and search console coverage.
- Archive final pre-launch database and uploads backup.
